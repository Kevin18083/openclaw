## 优化实施清单

### ✅ 已验证有效的优化点

1. **固定系统消息模板**
   - 使用统一的系统消息
   - 减少系统消息中的动态变量
   - 标准化 prompt 结构

2. **Prompt 前缀复用**
   - 将不变部分放在 prompt 开头
   - 动态内容后置
   - 使用 common prefix

3. **请求批处理**
   - 批量处理相似请求
   - 使用 multi-turn 对话模式
   - 减少单次请求数量

4. **缓存友好的 prompt 设计**
   - 减少随机/变化内容
   - 使用模板化 prompt
   - 合并相似请求

### 📋 实施步骤

#### 步骤 1: 更新系统消息配置
在 `openclaw.json` 中为每个使用场景定义固定的系统消息：
```json
{
  "systemPrompts": {
    "default": "你是一个专业的 AI 助手，名叫扎克，帮助用户解决问题。回答要简洁、准确、有用。",
    "coding": "你是一个资深软件工程师，编写高质量、可维护的代码。使用最佳实践，添加必要注释。",
    "analysis": "你是数据分析师，提供清晰的洞察和建议。用数据说话，避免模糊表述。"
  }
}
```

#### 步骤 2: 实现 Prompt 模板化
创建 prompt 模板文件，避免每次动态构建：
```javascript
// prompt-templates.js
const templates = {
  codeReview: `请审查以下代码：
1. 代码质量
2. 潜在问题
3. 改进建议

代码：
{code}`,

  dataAnalysis: `请分析以下数据：
1. 关键发现
2. 趋势分析
3. 建议行动

数据：
{data}`
};
```

#### 步骤 3: 启用请求批处理
将相似请求合并：
```javascript
// 优化前：多次单独请求
await callAI('问题 1');
await callAI('问题 2');
await callAI('问题 3');

// 优化后：批量请求
await callAI('请回答以下问题：\n1. 问题 1\n2. 问题 2\n3. 问题 3');
```

#### 步骤 4: 使用对话模式
利用 multi-turn 对话，保持上下文：
```javascript
// 优化前：每次都是新对话
await callAI({ messages: [{role: 'user', content: '问题 1'}] });
await callAI({ messages: [{role: 'user', content: '问题 2'}] });

// 优化后：延续对话
const conversation = [
  {role: 'user', content: '问题 1'},
  {role: 'assistant', content: '回答 1'},
  {role: 'user', content: '问题 2'}
];
await callAI({ messages: conversation });
```
