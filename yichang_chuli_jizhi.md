# 异常处理机制

## 概述
本文档定义扎克系统的异常处理流程和恢复机制。

## 异常分类

### A类异常（轻度异常）
- **表现**：临时性错误，可自动恢复
- **示例**：API调用超时、网络波动
- **处理**：自动重试，记录日志

### B类异常（中度异常）  
- **表现**：功能部分失效，需要干预
- **示例**：模型API不可用、配置文件错误
- **处理**：自动切换备用方案，通知用户

### C类异常（重度异常）
- **表现**：系统完全不可用
- **示例**：进程崩溃、数据损坏
- **处理**：触发恢复机制，紧急通知

## 恢复机制

### 1. 自动恢复流程
```
检测异常 → 分类评估 → 执行恢复 → 验证结果 → 记录日志
```

### 2. 恢复策略
- **API异常**：切换备用模型（阿里云 ↔ DeepSeek）
- **配置异常**：恢复最近备份
- **进程异常**：自动重启服务
- **数据异常**：从镜像恢复

### 3. 用户通知
- 轻度异常：仅记录日志
- 中度异常：操作时提示
- 重度异常：立即通知 + 恢复进度
