# JS 脚本 8 类注释规范

> 所有 JS 脚本必须遵循此注释规范，新增或修改时不能遗漏。

---

## 8 类注释标准

### 1. 文件头注释
说明脚本的目的、版本和核心功能。

```javascript
/**
 * 脚本名称 v1.0
 *
 * 功能说明：
 * 1. 功能 1 - 简要描述
 * 2. 功能 2 - 简要描述
 * 3. 功能 3 - 简要描述
 */
```

### 2. 配置说明
列出脚本中使用的配置项、常量、路径等。

```javascript
/**
 * 配置说明：
 * - CONFIG_PATH: 配置文件路径
 * - INTERVAL: 间隔时间 (ms)
 * - 状态枚举：pending, running, completed
 */
```

### 3. 用法示例
提供脚本的使用方法和命令示例。

```javascript
/**
 * 用法：
 *   node script.js "参数"          # 执行脚本
 *   node script.js status          # 查看状态
 *   node script.js help            # 显示帮助
 */
```

### 4. 示例输出
展示脚本运行时的预期输出格式。

```javascript
/**
 * 示例输出：
 *   ============================================================
 *   脚本名称
 *   ============================================================
 *   ✅ 操作成功
 *   📊 状态：正常
 */
```

### 5. 常见错误
列出可能出现的错误及解决方案。

```javascript
/**
 * 常见错误：
 * - 配置文件不存在 → 检查路径配置
 * - 权限不足 → 检查工作区权限
 * - JSON 解析失败 → 检查语法格式
 */
```

### 6. 依赖说明
说明 Node.js 版本和依赖模块。

```javascript
/**
 * 依赖：
 * - Node.js 14+
 * - fs, path (内置模块)
 * - xxx (外部模块，如有)
 */
```

### 7. 修改历史
记录版本变更和修改日期。

```javascript
/**
 * 修改历史：
 * - 2026-03-10: 初始版本
 * - 2026-03-11: 添加 8 类注释规范
 */
```

### 8. 关键函数注释 (JSDoc)
为每个函数添加 @param 和 @returns 注释。

```javascript
/**
 * 函数说明 - 简要描述函数功能
 * @param {类型} 参数名 - 参数描述
 * @param {类型} 参数名 - 参数描述
 * @returns {类型} 返回值描述
 */
function myFunction(param1, param2) {
  // 函数实现
}
```

 * - Node.js 14+

 * - 2026-03-10: 初始版本

 */

/**

if (require.main === module) {
  main(process.argv[2]);
