## 🔴 严重错误（影响系统稳定性）

### ERR-001: 备份目录不存在导致脚本崩溃
- **发生时间**: 2026-03-07 06:49
- **错误类型**: 代码错误
- **严重程度**: 🔴 严重
- **问题描述**: `cache-auto-check.js` 运行时，备份目录不存在导致脚本崩溃
- **根本原因**: 
  - 代码中虽然有创建目录的逻辑，但路径计算有误
  - `path.dirname(CONFIG.backupPath)` 返回的是 `backups` 而不是完整路径
- **解决方案**: 
  1. 手动创建目录：`New-Item -ItemType Directory -Force -Path "backups\cache-optimization"`
  2. 修复代码逻辑，确保目录存在
- **预防措施**: 
  - ✅ 所有文件操作前先检查目录是否存在
  - ✅ 使用 `fs.mkdirSync(path, { recursive: true })` 递归创建
  - ✅ 添加错误处理和友好的错误提示
- **状态**: ✅ 已解决
- **影响文件**: `cache-auto-check.js`

## 🟡 中等错误（影响功能但不崩溃）

### ERR-002: Gateway 配置重启后被重置
- **发生时间**: 2026-03-07 06:42
- **错误类型**: 配置错误
- **严重程度**: 🟡 中等
- **问题描述**: 修改 `openclaw.json` 添加缓存优化配置，重启后配置被重置
- **根本原因**: 
  - OpenClaw 的 Gateway 重启时会重新生成配置文件
  - 不支持的字段会被移除
  - `cacheOptimization` 不是标准配置字段
- **解决方案**: 
  - 使用独立的配置文件和系统（`cache-optimization-system.js`）
  - 不修改核心配置文件
- **预防措施**: 
  - ✅ 不修改 OpenClaw 核心配置文件
  - ✅ 使用独立的外部系统实现扩展功能
  - ✅ 先查阅文档确认配置字段是否支持
- **状态**: ✅ 已解决
- **影响文件**: `openclaw.json`

### ERR-003: Cron 任务 delivery 配置错误
- **发生时间**: 2026-03-07 06:54
- **错误类型**: 配置错误
- **严重程度**: 🟡 中等
- **问题描述**: 创建 cron 任务时添加 `delivery` 配置导致失败
- **根本原因**: 
  - `delivery` 配置仅支持 `sessionTarget="isolated"`
  - 使用 `sessionTarget="main"` 时不能添加 `delivery`
- **解决方案**: 
  - 移除 `delivery` 配置
  - 使用 `sessionTarget="main"` 时只用 `systemEvent`
- **预防措施**: 
  - ✅ 记住：`sessionTarget="main"` + `payload.kind="systemEvent"` 是标准组合
  - ✅ `delivery` 配置仅用于 `isolated` 会话
  - ✅ 创建 cron 前先确认配置规则
- **状态**: ✅ 已解决
- **影响文件**: cron 任务配置
