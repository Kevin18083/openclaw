## 📋 15 个流程详细说明

### 流程 1-5：基础测试

| 流程 | 检查内容 |
|------|----------|
| 1. 初步测试 | Node.js、Git 环境检查，5 轮重试机制 |
| 2. 复查验证 | 连续 3 次通过同一测试 |
| 3. 代码质量 | TODO/FIXME、硬编码密钥、console.log、未使用变量 |
| 4. 安全检查 | 敏感文件（.env、.pem）、SQL 注入、eval() 调用 |
| 5. 性能检查 | 嵌套循环 O(n²)、内存泄漏、异步阻塞 |

### 流程 6-11：工程化检查

| 流程 | 检查内容 |
|------|----------|
| 6. 文档兼容性 | README、CHANGELOG、LICENSE、跨平台支持 |
| 7. Git 提交 | 提交信息格式、工作区状态 |
| 8. 环境变量 | .env 文件、NODE_ENV 配置 |
| 9. 依赖健康 | package-lock.json、go.sum、过期依赖 |
| 10. 单元测试 | 自动运行测试套件、覆盖率检测 |
| 11. 构建检查 | 构建成功、构建后测试 |

### 流程 12-14：专项领域检查

| 流程 | 检查内容 |
|------|----------|
| 12. 数据库迁移 | 迁移文件、SQL 语法、回滚脚本 |
| 13. 回滚灾难 | 回滚脚本、备份配置、灾难文档 |
| 14. 日志检查 | 日志配置、脱敏、轮转配置 |

### 流程 15-17：质量提升检查

| 流程 | 检查内容 |
|------|----------|
| 15. 错误处理 | try-catch、Promise.catch、错误边界 |
| 16. 监控告警 | Prometheus、Grafana、健康端点、告警规则 |
| 17. 生成报告 | 汇总所有测试结果，输出最终判断 |
