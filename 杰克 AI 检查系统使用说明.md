# 杰克 AI 检查系统使用说明

> **版本**: v3.1
> **创建时间**: 2026-03-11
> **主用方案**: DeepSeek API（¥0.001/1K tokens，非常便宜）

---

## 🎯 核心设计

**主用方案：DeepSeek API**
- ✅ 已配置，立即可用
- ✅ 成本低廉（¥0.001/1K tokens）
- ✅ 响应快速（1-3 秒）
- ✅ 效果优秀

**备用方案：规则检查**
- 当 API 不可用时自动降级
- 基础规则检查
- 无需网络

---

## 🔄 完整流程

```
扎克提交任务
    ↓
Hook 自动触发 DeepSeek AI 检查
    ↓
杰克 AI 读取扎克提交的内容
    ↓
DeepSeek 输出具体检查意见
    ↓
初检通过 → 进入深检
    ↓
DeepSeek 再次深度检查
    ↓
深检通过 → 自动归档
    ↓
输出"杰克已阅" → 扎克向罗总汇报
```

---

## 🔧 使用命令

### 创建检查任务
```bash
node jack-review.js "让杰克检查 [任务名]"
```

### 扎克提交内容
```bash
node jack-review.js submit "[任务详情]"
```

### 杰克 AI 初检（自动）
```bash
# Hook 自动触发，无需手动执行
# 或手动执行：
node jack-review.js initial ai
```

### 杰克 AI 深检（自动）
```bash
# 初检通过后自动触发
# 或手动执行：
node jack-review.js deep ai
```

### 查看状态
```bash
node jack-review.js status
```

## ✅ 检查清单

### 初检（5 项）
1. 格式检查 - 报告格式是否完整
2. 步骤检查 - 步骤是否清晰、可追溯
3. 文件检查 - 修改的文件是否存在
4. 语法检查 - 代码/配置语法是否正确
5. 完整性检查 - 是否有遗漏

### 深检（5 项）
1. 逻辑检查 - 代码逻辑是否正确
2. 完整性检查 - 功能是否完整实现
3. 最佳实践 - 是否符合编码规范
4. 可维护性 - 代码是否易于维护
5. 性能考虑 - 是否有性能问题

## 🔒 防伪造机制

### 1. AI 必须读取内容
- DeepSeek 提示词包含扎克提交的完整内容
- AI 必须分析后才能输出结果

### 2. 检查结果存档
- 每次检查结果保存到 `.jack-checks/`
- 包含 AI 原始响应

### 3. 任务历史完整
- 每次检查记录到任务 history
- 包含检查时间、结果、意见

### 4. 罗总可以验证
- 查看 `.jack-checks/` 目录确认 AI 检查过
- 查看任务历史确认流程完整
