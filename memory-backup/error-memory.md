### 错误 001：Thrift 验证子 Shell 变量不生效

**发现日期**：2026-03-09

**来源文件**：`idl.yaml` (Thrift 语法验证工作流)

**错误现象**：
```bash
find "$GITHUB_WORKSPACE/idl" -name '*.thrift' -print0 | while read -d '' file; do
  if ! kitex ... "$file"; then
    ERROR_FOUND=1  # ❌ 设置了错误标志
  fi
done

# 后面检查时 ERROR_FOUND 永远是 0！
if [ $ERROR_FOUND -eq 1 ]; then exit 1; fi  # ❌ 永远不执行
```

**根本原因**：
- `while` 循环通过管道运行，在子 shell 中执行
- 子 shell 中的变量修改不影响父 shell
- 即使有错误，脚本也会成功退出（exit 0）

**解决方案**：
```bash
# 方法 1：用临时文件记录错误
ERROR_FILE=$(mktemp)
find "$GITHUB_WORKSPACE/idl" -name '*.thrift' -print0 | while read -d '' file; do
  if ! kitex ... "$file"; then
    echo "error" > "$ERROR_FILE"
  fi
done
if [ -s "$ERROR_FILE" ]; then exit 1; fi

# 方法 2：直接退出
find "$GITHUB_WORKSPACE/idl" -name '*.thrift' -print0 | while read -d '' file; do
  if ! kitex ... "$file"; then
    echo "IDL gen code error in file: $file"
    rm -rf "$TEMP_DIR"
    exit 1  # 直接退出整个脚本
  fi
done
```

**学习收获**：
- ⚠️ Bash 管道创建子 shell
- ⚠️ 子 shell 变量不影响父 shell
- ✅ 用临时文件或直接退出来传递错误状态

**如何避免**：
- 写 `find | while` 循环时，注意变量作用域
- 需要传递状态时，用临时文件或返回值

**相关测试**：
```bash
# 测试框架中已加入此模式检查
test_bash_subshell() {
    log_info "测试子 shell 变量传递"

    # 错误示例（会失败）
    local count=0
    echo "1 2 3" | while read i; do
        count=$((count + 1))
    done
    # count 还是 0！

    # 正确示例
    local count=0
    while read i; do
        count=$((count + 1))
    done < <(echo "1 2 3")
    # count = 3 ✅
}
```

### 错误 002：Bash 数组语法错误（多余逗号）

**发现日期**：2026-03-09

**来源文件**：`check-file-size.sh` (文件体积检查脚本)

**错误现象**：
```bash
EXCLUDE_PATTERNS=(
  'apps/fornax/**',  # ❌ 多了逗号
  "packages/arch/semi-theme-hand01",  # ❌ 多了逗号
)
```

**根本原因**：
- Bash 数组元素之间应该用空格或换行分隔
- 逗号会被当作字面值，导致排除模式匹配失败

**解决方案**：
```bash
EXCLUDE_PATTERNS=(
  'apps/fornax/**'   # ✅ 没有逗号
  "packages/arch/semi-theme-hand01"  # ✅ 没有逗号
)
```

**学习收获**：
- ⚠️ Bash 数组不用逗号分隔
- ⚠️ 逗号会成为模式的一部分

**如何避免**：
- 写 Bash 数组时，元素之间用空格或换行
- 不用逗号（除非逗号是模式的一部分）

### 错误 003：MySQL 端口检测不可靠

**发现日期**：2026-03-09

**来源文件**：`ci@backend.yml` (后端测试 CI)

**错误现象**：
```bash
for i in {1..60}; do
  if cat /proc/net/tcp | grep 0CEA; then
    echo "MySQL port 3306 is listening!"
    break
  fi
  sleep 1
done
```

**根本原因**：
1. `/proc/net/tcp` 是 16 进制，`0CEA` = 3306，但可能有多个端口匹配
2. 没有检查 MySQL 是否真的能连接

**解决方案**：
```bash
for i in {1..60}; do
  if mysql -h 127.0.0.1 -P 3306 -u root -proot -e "SELECT 1" &>/dev/null; then
    echo "MySQL is ready!"
    break
  fi
  echo "Waiting for MySQL... ($i/60)"
  sleep 1
done
```

**学习收获**：
- ⚠️ 端口监听不代表服务可用
- ✅ 直接连接测试最可靠

**如何避免**：
- 检测服务时，用实际连接测试
- 不要只检查端口监听

### 错误 004：Go 测试 coverpkg 有多余逗号

**发现日期**：2026-03-09

**来源文件**：`ci@backend.yml`

**错误现象**：
```bash
coverpkg=$module"/...,"$coverpkg
# 累加后：module1/...,module2/...,module3/...,
# 最后多一个逗号！
go test -coverpkg=$coverpkg $list
```

**根本原因**：
- 累加时每次都加逗号
- 最后没有去掉末尾逗号

**解决方案**：
```bash
# 方法 1：去掉末尾逗号
coverpkg=${coverpkg%,}
go test -coverpkg="$coverpkg" ...

# 方法 2：用数组收集
declare -a modules
for module in $modules; do
  modules+=("$module/...")
done
coverpkg=$(IFS=,; echo "${modules[*]}")
```

**学习收获**：
- ⚠️ 字符串累加分隔符时，注意末尾多余符号
- ✅ 用数组收集再连接更安全

### 错误 005：Git 钩子权限问题

**发现日期**：2026-03-09

**来源文件**：测试框架使用示例

**错误现象**：
```bash
./test-script.sh
# bash: ./test-script.sh: Permission denied
```

**根本原因**：
- 脚本没有执行权限

**解决方案**：
```bash
chmod +x ./test-script.sh
./test-script.sh
```

**学习收获**：
- ⚠️ Bash 脚本需要执行权限
- ✅ 创建脚本后，先 `chmod +x`

**如何避免**：
- 测试框架中已加入自动添加权限：
```bash
if [ ! -x "$script_path" ]; then
    chmod +x "$script_path"
fi
```

### 错误 006：日志目录不存在

**发现日期**：2026-03-09

**来源文件**：测试框架开发

**错误现象**：
```bash
echo "log" >> ./test-logs/main.log
# bash: ./test-logs/main.log: No such file or directory
```

**根本原因**：
- 日志目录未创建

**解决方案**：
```bash
mkdir -p "$LOG_DIR"
```

**学习收获**：
- ⚠️ 写日志前先创建目录
- ✅ 用 `mkdir -p` 确保目录存在

**如何避免**：
- 测试框架初始化时自动创建：
```bash
init() {
    mkdir -p "$LOG_DIR"
    echo "初始化完成"
}
```
