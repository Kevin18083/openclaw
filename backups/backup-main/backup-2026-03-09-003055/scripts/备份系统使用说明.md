##  系统概述

备份系统提供**3 层自动保护**，确保数据安全：

### 优先级架构

```
┌─────────────────────────────────────┐
│  主备份 (C 盘 1 号)   ← 默认使用    │
│  backups\knowledge-main\            │
└─────────────────────────────────────┘
              ↓ 故障自动切换
┌─────────────────────────────────────┐
│  备备份 (C 盘 2 号)   ← 自动备用    │
│  backups\knowledge-backup\          │
└─────────────────────────────────────┘
              ↓ 双 C 盘故障
┌─────────────────────────────────────┐
│  杀手锏 (D 盘)       ← 最后防线    │
│  D:\AAAAAA\ClaudeBackups\backup\    │
└─────────────────────────────────────┘
```

---

## 🔄 自动切换逻辑

### 工作流程

1. **每次备份执行前** → 自动健康检查
2. **主备份健康** → 继续使用主备份
3. **主备份故障** → 自动切换到备备份
4. **主备份恢复** → 自动切回主备份
5. **双 C 盘故障** → 启用 D 盘杀手锏

### 健康检查标准

- ✅ 目录存在
- ✅ 可访问（能读取目录列表）
- ✅ 关键项目 ≥ 75%（4 个关键项目至少 3 个存在）

**关键项目清单**:
- `memory/` - 记忆目录
- `skills/` - 技能目录
- `AGENTS.md` - 代理配置
- `MEMORY.md` - 长期记忆

## 📋 使用命令

### 自动检查（推荐）

```bash
# 运行自动备份（自动包含备份检查）
node scripts/auto-backup.js

# 或单独检查备份
node scripts/backup-switcher.js check
```

### 查看状态

```bash
# 查看备份系统完整状态
node scripts/backup-switcher.js status
```

### 获取当前备份路径

```bash
# 获取当前使用的备份路径
node scripts/backup-switcher.js path
```

### 强制切换（手动）

```bash
# 切换到主备份
node scripts/backup-switcher.js switch primary

# 切换到备备份
node scripts/backup-switcher.js switch backup

# 切换到 D 盘杀手锏
node scripts/backup-switcher.js switch disaster
```

## 🔧 集成说明

### 已集成到

1. **自动备份脚本** (`auto-backup.js`)
   - 每次备份前自动检查
   - 自动切换故障备份
   - 记录到状态文件

2. **心跳检查** (HEARTBEAT.md)
   - 每日 00:00 自动运行备份
   - 包含备份健康检查

### 调用示例

```javascript
const BackupSwitcher = require('./backup-switcher.js');

// 自动检查并切换
const state = BackupSwitcher.autoSwitch();
console.log('当前备份:', state.currentBackup);

// 获取当前备份路径
const path = BackupSwitcher.getCurrentBackupPath();

// 强制切换
BackupSwitcher.forceSwitch('backup');
```

## ⚠️ 故障处理

### 所有备份都故障

如果三个备份都不健康：
1. 系统会记录错误到 `backup-state.json`
2. 备份系统仍然可以尝试运行
3. 需要手动修复或重建备份

### 恢复步骤

```bash
# 1. 检查状态
node scripts/backup-switcher.js status

# 2. 手动修复备份目录

# 3. 重新运行检查
node scripts/backup-switcher.js check
```

## 🎯 设计原则

1. **自动化优先** - 无需人工干预
2. **快速故障检测** - 每次备份前检查
3. **透明切换** - 记录所有切换原因
4. **优先本地** - C 盘优先，D 盘最后
5. **自动恢复** - 主备份恢复后自动切回
