# 杰克测试框架 v8.0 - 三种模式优化版

> 版本：v8.0 | 更新日期：2026-03-09 | 状态：✅ 完成

---

## 🚀 优化方案

解决原有完整流程耗时过长（14-16 分钟）的问题，提供**三种测试模式**：

| 模式 | 流程数 | 耗时 | 适用场景 |
|------|--------|------|----------|
| **快速模式** | 5 个 | 1 分钟 | 开发中快速验证 |
| **标准模式** | 10 个 | 5 分钟 | 提交前检查 |
| **完整模式** | 15 个 | 15 分钟 | 实装前验证 |

---

## 📋 三种模式详细对比

### 快速模式（5 流程 - 1 分钟）

```bash
bash unified-gateway.sh --quick .
```

| 流程 | 检查内容 |
|------|----------|
| 1. 初步测试 | 环境、Git 检查 |
| 2. 代码质量 | TODO、密钥、调试语句 |
| 3. 安全检查 | 敏感文件、SQL 注入 |
| 4. 依赖健康 | package-lock、go.sum |
| 5. 日志检查 | 日志配置、脱敏 |

**适用场景**：
- ✅ 开发过程中快速验证
- ✅ 每次保存文件后
- ✅ Git commit 前快速检查

---

### 标准模式（10 流程 - 5 分钟）⭐ 推荐

```bash
bash unified-gateway.sh --standard .
```

| 流程 | 检查内容 |
|------|----------|
| 1-5. 基础测试 | 同快速模式 + 性能检查 |
| 6. 文档检查 | README、CHANGELOG |
| 7. 环境变量 | .env 配置 |
| 8. 依赖健康 | 锁文件、过期依赖 |
| 9. 单元测试 | 自动运行测试 |
| 10. 日志检查 | 日志完整性 |

**适用场景**：
- ✅ Git 提交前
- ✅ 每日构建
- ✅ PR 提交前

---

### 完整模式（15 流程 - 15 分钟）

```bash
bash unified-gateway.sh --full .
```

| 阶段 | 流程 | 检查内容 |
|------|------|----------|
| 基础测试 | 5 个 | 环境、代码质量、安全、性能 |
| 工程化检查 | 6 个 | 文档、Git、依赖、单元测试、构建 |
| 专项领域 | 3 个 | 数据库迁移、回滚灾难、日志 |
| 质量提升 | 2 个 | 错误处理、监控告警 |

**适用场景**：
- ✅ 实装前最终验证
- ✅ 发布前检查
- ✅ 重要功能提交

## 🎯 使用示例

### 开发中（频繁使用）

```bash
# 每隔几分钟运行一次，快速验证
bash unified-gateway.sh --quick .

# 或者使用快捷键/脚本
alias tq='bash unified-gateway.sh --quick'
tq .
```

### 提交前（每次 commit）

```bash
# 提交前运行标准检查
bash unified-gateway.sh --standard .

# 配置 Git alias
git config --global alias.st '!'unified-gateway.sh --standard
git st .
```

### 实装前（重要节点）

```bash
# 实装前运行完整检查
bash unified-gateway.sh --full .

# 查看完整报告
cat test-logs/summary-*.log
```

## ⚙️ 配置选项

可以在 `.testrc` 或环境变量中配置：

```bash
# 默认模式（推荐 standard）
MODE=standard

# 测试超时时间（秒）
TEST_TIMEOUT=30      # 单项测试超时
BUILD_TIMEOUT=120    # 构建超时

# 重试次数
MAX_RETRIES=2        # 快速模式重试
VERIFY_RETRIES=2     # 复查验证轮数
```

## 🎯 推荐工作流

### 日常开发

```
开发功能 → --quick 检查 → 修复问题 → --quick 检查 → 通过
                                        ↓
                                   git commit
                                        ↓
                                 --standard 检查
                                        ↓
                                   git push
                                        ↓
                              (GitHub Actions 运行完整检查)
```

### 实装前

```
完成开发 → --standard 检查 → 通过
                              ↓
                         --full 检查
                              ↓
                    所有检查通过 → 实装
```

## 📝 Git Hook 集成

可以在 `.hooks/pre-commit` 中使用快速模式：

```bash
#!/bin/bash
# .hooks/pre-commit

echo "运行快速检查..."
bash unified-gateway.sh --quick .

if [ $? -ne 0 ]; then
    echo "❌ 快速检查失败，阻止提交"
    exit 1
fi

echo "✅ 快速检查通过"
exit 0
```
