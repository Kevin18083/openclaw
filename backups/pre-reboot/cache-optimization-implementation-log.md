### 1. 固定系统消息 ✅
**3 个标准模板**:

```

【代码专家】

```

**使用规则**:
- 通用对话 → 使用"通用助手"
- 代码相关 → 使用"代码专家"
- 数据分析 → 使用"数据分析师"

### 2. Prompt 模板化 ✅
**15 个标准模板**:

#### 基础问答（5 个）
1. weather - 天气查询
2. fact - 事实查询
3. calculation - 计算问题
4. definition - 定义解释
5. list - 列表生成

#### 中等难度（5 个）
6. codeGeneration - 代码生成
7. dataAnalysis - 数据分析
8. textSummary - 文本摘要
9. comparison - 对比分析
10. problemSolving - 问题解决

#### 高难度（5 个）
11. systemDesign - 系统设计
12. complexCode - 复杂代码
13. reasoning - 多步推理
14. debug - 调试分析
15. creative - 创意写作

**使用规则**: 所有请求必须使用模板，不自己构建 prompt

---

### 3. 前缀复用 ✅
**固定前缀格式**:

```
请按照以下要求回答：1) 简洁明了 2) 准确专业 3) 实用可操作

[具体问题]
```

**使用规则**:
- 不变内容放在 prompt 开头
- 动态内容后置
- 保持结构一致

---

### 4. 请求批处理 ✅
**批处理策略**:

```
相似请求合并:
- 多个相关问题 → 一次请求
- 连续对话 → 使用 multi-turn 模式
- 批量处理 → 5 个请求/批
```

**使用规则**:
- 相似问题合并发送
- 使用对话模式而非单次请求
- 批处理大小：5

## 📋 使用指南

### 立即开始使用

**1. 使用固定系统消息**
```javascript

// 代码相关
const systemPrompt = '你是一个资深软件工程师，编写高质量、可维护的代码。使用最佳实践，添加必要注释。';

// 数据分析
const systemPrompt = '你是数据分析师，提供清晰的洞察和建议。用数据说话，避免模糊表述。';
```

**2. 使用 Prompt 模板**
```javascript
const { TemplateLibrary } = require('./cache-optimization-system');

// 选择模板
const template = TemplateLibrary.basic.weather;

// 应用模板
const prompt = template.template.replace('{location}', '北京');
```

**3. 使用前缀复用**
```javascript

const prompt = commonPrefix + userQuestion;
```

## 📞 故障排查

### 问题 1: 命中率没有提升
**检查**:
- 是否使用了固定系统消息？
- 是否使用了 Prompt 模板？
- 系统消息是否频繁变化？

**解决**:
- 严格使用 3 个标准系统消息
- 所有请求使用 15 个模板
- 减少动态变化

### 问题 2: 响应时间变长
**检查**:
- 批处理大小是否合理？
- 模板是否过于复杂？

**解决**:
- 调整批处理大小（3-5 个）
- 简化模板结构

### 问题 3: 系统异常
**检查**:
- 查看错误日志
- 检查配置文件

**解决**:
- 运行 `cache-auto-check.js`
- 查看 `cache-optimization-report.md`
