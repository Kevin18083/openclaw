## 🧠 系统组成

### 1. 错误日志 (`error-log.md`)
- 记录所有错误、异常、问题
- 分析根本原因
- 制定预防措施
- 跟踪解决状态

### 2. 错误监控 (`error-monitor.js`)
- 自动记录错误
- 检查类似错误
- 生成统计报告
- 追踪重复错误

### 3. 预防清单 (`prevention-checklist.md`)
- 操作前检查清单
- 分类最佳实践
- 历史错误参考
- 持续更新优化

### 4. 学习记忆 (`MEMORY.md` + `memory/`)
- 记录重要经验
- 存储关键教训
- 定期回顾整理
- 形成长期记忆

## 📊 错误分类

### 按类型
1. **代码错误** - 程序 bug、逻辑错误
2. **配置错误** - 配置不当、参数错误
3. **逻辑错误** - 思考不周、判断失误
4. **操作错误** - 操作失误、流程错误
5. **沟通错误** - 理解偏差、表达不清

### 按严重程度
1. **🔴 严重** - 系统崩溃、数据丢失
2. **🟡 中等** - 功能异常、体验受损
3. **🟢 轻微** - 小问题、不影响使用

## 🧩 知识沉淀

### 经验库
将错误和解决方案整理成经验：
```markdown
## 经验编号：EXP-001
**主题**: 文件操作前检查目录存在
**来源**: ERR-001 备份目录不存在
**应用场景**: 所有文件写入操作
**最佳实践**: 
1. 使用 fs.existsSync() 检查
2. 使用 fs.mkdirSync(path, { recursive: true }) 创建
3. 添加 try-catch 错误处理
```

### 模式库
识别常见错误模式：
```markdown
## 模式：目录不存在
**特征**: 文件写入失败、路径错误
**原因**: 未检查/创建父目录
**解决**: 递归创建目录
**预防**: 加入检查清单
```

## 📞 快速参考

### 遇到错误时
1. 记录到 `error-log.md`
2. 运行 `error-monitor.js` 分析
3. 查看 `prevention-checklist.md`
4. 制定预防和解决方案

### 重要操作前
1. 查看 `prevention-checklist.md`
2. 检查 `error-log.md` 有无类似错误
3. 按清单逐项检查
4. 确认无误后执行
