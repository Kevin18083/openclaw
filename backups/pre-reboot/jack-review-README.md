# 扎克操作 - 杰克检查 自动化系统

> **版本**: v1.0
> **创建时间**: 2026-03-09
> **目标**: 自动化执行检查流程，无需手动操作

---

## 🚀 快速开始

### 触发检查（罗总）

```bash
# 方式 1: 直接说任务名（包含触发词）
node jack-review.js "让杰克检查缓存配置"
node jack-review.js "给杰克检查新脚本"

# 方式 2: 使用 trigger 命令
node jack-review.js trigger "任务名称"
```

### 扎克提交

```bash
node jack-review.js submit "任务详情..."
```

### 杰克初检

```bash
node jack-review.js initial '{"passed":false,"issues":[...],"suggestions":[...]}'
```

### 杰克深检

```bash
node jack-review.js deep '{"passed":true}'
```

### 查看状态

```bash
node jack-review.js status
```

## 🔧 命令说明

### 1. 触发检查

```bash
node jack-review.js "让杰克检查 [任务名]"
```

**输出**：
- 创建任务文件
- 显示扎克需要提交的内容
- 提示下一步操作

---

### 2. 扎克提交

```bash
node jack-review.js submit "任务详情"
```

**提交内容格式**：
```markdown
## 任务名称
[任务描述]

## 执行步骤
1. [步骤 1]
2. [步骤 2]

## 操作内容
- 修改的文件：[路径]
- 新增的内容：[内容]

## 自检查结果
- [x] 语法正确
- [x] 逻辑正确
```

---

### 3. 杰克初检

```bash
node jack-review.js initial '{"passed":false,"issues":[...],"suggestions":[...]}'
```

**参数说明**：
- `passed`: true/false
- `issues`: 问题列表 `[{description, location, severity}]`
- `suggestions`: 修改建议列表

---

### 4. 杰克深检

```bash
node jack-review.js deep '{"passed":true}'
```

**参数说明**：同初检

---

### 5. 查看状态

```bash
node jack-review.js status
```

**输出**：
- 当前活跃任务列表
- 每个任务的状态和修订次数

---

## 📊 任务状态

| 状态 | 说明 |
|------|------|
| `pending` | 待提交 |
| `initial_review` | 初检中 |
| `modifying` | 修改中 |
| `deep_review` | 深检中 |
| `passed` | 已通过 |
| `archived` | 已归档 |

## 📝 使用示例

### 完整流程示例

**1. 触发检查**
```bash
node jack-review.js "让杰克检查缓存配置"
```

**2. 扎克提交**
```bash
node jack-review.js submit "缓存配置已修改，启用了 promptCache，TTL 设置为 7200 秒"
```

**3. 杰克初检（有问题）**
```bash
node jack-review.js initial '{"passed":false,"issues":[{"description":"JSON 缺少逗号","location":"第 5 行","severity":"高"}],"suggestions":["在第 5 行添加逗号"]}'
```

**4. 扎克修改后重新提交**
```bash
node jack-review.js submit "已添加逗号，请重新检查"
```

**5. 杰克初检通过，进入深检**
```bash
node jack-review.js initial '{"passed":true}'
```

**6. 杰克深检通过**
```bash
node jack-review.js deep '{"passed":true}'
```

**7. 扎克向罗总汇报**
```markdown
## 📋 任务完成报告

**任务名称**: 缓存配置修改
**执行时间**: 2026/3/9 23:53:53 - 2026/3/9 23:59:00
**检查状态**: ✅ 杰克两轮检查通过
**迭代次数**: 1 次

### 执行内容
修改缓存配置，启用 promptCache

### 修改的文件
- cache-config.json

请罗总审阅。
```

## 🎯 自动化特性

| 特性 | 说明 |
|------|------|
| 自动创建任务 | 触发时自动生成任务文件和 ID |
| 状态跟踪 | 自动更新任务状态 |
| 修订计数 | 自动记录修改次数 |
| 历史日志 | 自动记录所有操作历史 |
| 自动归档 | 通过后自动移到归档目录 |
| 汇报格式 | 自动输出汇报模板 |

---

## 📌 注意事项

1. **任务文件保存在** `.jack-review-tasks/` 目录
2. **归档文件保存在** `.jack-review-archive/` 目录
3. **每次检查都有独立任务 ID**，便于追踪
4. **修订次数自动累加**，便于了解迭代情况
