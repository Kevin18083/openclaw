## 📦 什么是缓存优化器？

缓存优化器帮助你**提高阿里云 API 的缓存命中率**，从而：
- ✅ **更快响应** - 命中缓存的请求响应速度提升 10-20%
- ✅ **节省 Token** - 减少 15-25% 的 Token 消耗
- ✅ **降低成本** - 缓存读取比重新计算便宜

## 📖 使用方法

### 方法 1: 在脚本中使用（推荐）

```javascript
const { generateOptimizedRequest } = require('./zach-cache-optimizer');

// 生成优化的请求
const request = generateOptimizedRequest(
  'code',           // 类别：basic, code, analysis, learn
  'func',           // 模板名
  {                 // 变量
    functionality: '计算斐波那契数列',
    language: 'JavaScript',
    inputs: 'n (数字)',
    outputs: '第 n 个斐波那契数'
  },
  'coder'          // 系统消息类型：default, coder, analyst, chat
);

// 使用 request.system 和 request.user 发送请求
console.log('系统消息:', request.system);
console.log('用户消息:', request.user);
```

### 方法 2: 直接使用模板

```javascript
const { TEMPLATES, applyTemplate } = require('./zach-cache-optimizer');

// 选择模板
const template = TEMPLATES['weather'];

// 应用变量
const prompt = applyTemplate(template, {
  location: '北京',
  time: '今天'
});

console.log(prompt);
```

## 💡 提高缓存命中的技巧

### 1. 固定系统消息 ✅

```javascript
// ✅ 好 - 始终使用相同的系统消息
const req1 = generateOptimizedRequest('code', 'func', {...}, 'coder');
const req2 = generateOptimizedRequest('code', 'func', {...}, 'coder');

// ❌ 坏 - 系统消息变来变去
const req1 = generateOptimizedRequest('code', 'func', {...}, 'coder');
const req2 = generateOptimizedRequest('code', 'func', {...}, 'default'); // 不同！
```

### 2. 使用模板 ✅

```javascript
// ✅ 好 - 使用模板，前缀固定
const req = generateOptimizedRequest('basic', 'weather', {
  location: '北京',
  time: '今天'
});

// ❌ 坏 - 自由发挥，每次都不一样
const prompt = `你好，请告诉我北京今天的天气情况，谢谢！`;
```

### 3. 批处理相似请求 ✅

```javascript
// ✅ 好 - 连续发送相似请求
for (let i = 0; i < 5; i++) {
  generateOptimizedRequest('basic', 'fact', {
    question: `问题${i}`
  }, 'default');
}

// ❌ 坏 - 请求间隔太久，缓存过期
```

### 4. 变量位置固定 ✅

```javascript
// ✅ 好 - 变量在模板中的位置固定
// 模板：请提供{location}的天气信息

// ❌ 坏 - 变量位置变来变去
// 请求 1: 请提供北京的天气信息
// 请求 2: 天气信息，北京的，请提供
```

## 🔍 常见问题

### Q: 为什么命中率还是低？

**A**: 可能原因：
1. 没有使用模板，自由发挥太多
2. 系统消息经常变化
3. 请求间隔超过缓存 TTL（2 小时）
4. 任务本身多样性太高（创意、写作等）

### Q: 如何选择合适的模板？

**A**: 参考这个流程图：
```
任务类型？
├─ 查询事实/数据 → basic (weather, fact, calc, define, list)
├─ 写代码/调试 → code (func, debug, refactor, review, api)
├─ 分析数据/报告 → analysis (data, compare, swot, summary, root)
└─ 学习新知识 → learn (concept, compare2, roadmap, best)
```

### Q: 可以自定义模板吗？

**A**: 可以！在 `zach-cache-optimizer.js` 的 `TEMPLATES` 对象中添加：
```javascript
const TEMPLATES = {
  'myCustom': {
    name: '我的模板',
    prefix: PROMPT_PREFIXES.common,
    template: `我的模板内容：{variable}`,
    category: 'basic'
  }
};
```

## 📈 效果对比

- 系统消息：固定 4 种类型

- 结果：缓存命中高，Token 消耗低
