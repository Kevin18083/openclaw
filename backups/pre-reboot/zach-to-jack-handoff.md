# 扎克 → 杰克 交接流程

> **版本**: v1.0
> **创建时间**: 2026-03-09
> **目标**: 扎克完成任务后，如何提交给杰克检查

---

## 📋 交接方式

### 方式 1: 你（罗总）通知杰克

**最简单直接**

```
罗总 → 杰克：扎克完成了 [任务名]，你来检查一下
```

**杰克收到后**：
1. 回复：收到，正在检查
2. 进行初检 → 深检
3. 输出检查结果
4. 通知扎克和你

---

### 方式 2: 扎克提交交接文件

扎克完成任务后，创建交接文件：

**文件位置**：
```
C:\Users\17589\.openclaw\workspace\.pending-review\
```

**文件名格式**：
```
[日期]-[任务名]-待检查.md
例如：20260309-cache-config-待检查.md
```

**文件内容格式**：
```markdown
# 待检查 - [任务名称]

**提交人**: 扎克
**提交时间**: 2026-03-09 20:00
**任务描述**: [简要说明]

---

## 执行步骤
1. [步骤 1]
2. [步骤 2]
3. [步骤 3]

## 操作内容
- 修改的文件：[路径]
- 新增的内容：[内容]
- 配置变更：[详情]

## 自检查结果
- [x] 语法正确
- [x] 逻辑正确
- [x] 无遗漏步骤

---

## 📋 请杰克检查

- [ ] 初检
- [ ] 深检

**提交给**: @杰克
```

---

### 方式 3: 自动触发检查

创建一个脚本，扎克完成后运行：

```bash
# 扎克完成任务后运行
node submit-for-review.js "任务名称" "任务描述"
```

脚本会自动：
1. 生成交接文件
2. 发送通知（如配置了通知系统）
3. 等待杰克检查

## 🎯 推荐做法

### 简单场景（小任务）
```
扎克完成 → 你告诉我 → 我检查 → 输出结果
```

### 复杂场景（大任务）
```
扎克完成 → 创建交接文件 → 你通知我 → 我检查 → 输出结果 → 扎克汇报
```

### 自动化场景（配置了通知）
```
扎克完成 → 运行提交脚本 → 自动创建文件 + 通知 → 我检查 → 输出结果
```

## ️ 文件管理

### 目录结构
```
workspace/
├── .pending-review/        # 待检查文件
│   ├── 20260309-task1-待检查.md
│   └── 20260309-task2-待检查.md
├── .reviews/               # 已完成检查
│   ├── 20260309-task1-已通过.md
│   └── 20260309-task2-需修改.md
```

### 文件流转
```
扎克创建 → .pending-review/
         ↓
    杰克检查
         ↓
    已通过 → 移到 .reviews/ + 通知扎克汇报
    需修改 → 移到 .reviews/ + 通知扎克修改
```

## 📌 使用说明

### 扎克
1. 完成任务
2. 创建交接文件（或使用提交脚本）
3. 等待杰克检查
4. 如有问题，修改后重新提交
5. 检查通过后，向罗总汇报

### 杰克
1. 收到通知（或查看 .pending-review/ 目录）
2. 读取交接文件
3. 进行初检和深检
4. 在文件中填写检查结果
5. 通知扎克结果

### 罗总
1. 接收扎克完成通知
2. 通知我检查
3. 等待检查完成
4. 接收扎克最终汇报

## 🚀 快速开始

### 扎克提交检查（推荐方式）

**步骤 1**: 创建目录（首次使用）
```bash
mkdir workspace/.pending-review
```

**步骤 2**: 创建交接文件
```bash
# 复制模板
copy workspace\.template-pending-review.md workspace\.pending-review\20260309-任务名 - 待检查.md
```

**步骤 3**: 填写内容并保存

**步骤 4**: 通知
```
扎克：罗总，我完成了 [任务名]，交接文件已创建
罗总：好的，杰克你来检查一下
杰克：收到，正在检查...
```

*此流程确保扎克和杰克的交接顺畅，信息不丢失*
