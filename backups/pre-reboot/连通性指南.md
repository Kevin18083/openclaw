# 杰克测试框架 - 连通性指南

> 版本：v1.0 | 创建日期：2026-03-09 | 三大功能：快速检查 + 决策树 + 记忆库回顾

---

## 🎯 概述

**连通性** = 快速检查 + 决策树 + 记忆库回顾 三者联动

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   记忆库回顾 ──→ 快速检查 ──→ 决策树 ──→ 实装              │
│       ↓              ↓           ↓                          │
│   避免犯错       1 分钟出结果   评分决策                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 🚀 快速开始

### 方式 1：统一入口（推荐）

```bash
# 运行完整流程（包含所有检查）
bash unified-gateway.sh --full ./项目路径

# 示例
bash unified-gateway.sh --full .
```

### 方式 2：单独运行

```bash
# 只运行记忆库回顾
bash unified-gateway.sh --memory .

# 只运行快速检查
bash unified-gateway.sh --quick .

# 只运行决策树评估
bash unified-gateway.sh --decision .
```

### 方式 3：直接调用脚本

```bash
# 记忆库回顾
bash memory-review.sh

# 快速检查
bash quick-check.sh

# 决策树评估
bash decision-tree.sh

# 完整测试流水线
bash test-framework-plus.sh
```

## 🔄 连通性流程

### 完整流程（4 个阶段）

```
阶段 1/4: 记忆库回顾
   ↓
   检查历史错误模式
   避免重复犯错
   ↓
阶段 2/4: 快速检查
   ↓
   5 个核心检查（1 分钟）
   语法 → 安全 → 密钥 → 测试 → 构建
   ↓
阶段 3/4: 决策树评估
   ↓
   权重评分（满分 100）
   ≥80 分且关键项通过 → 可以实装
   ↓
阶段 4/4: 完整测试流水线
   ↓
   14 个流程完整测试
   生成最终报告
```

### 各阶段详解

#### 阶段 1: 记忆库回顾
**作用**：查看历史错误，避免重复犯错

```bash
bash memory-review.sh ./项目路径
```

#### 阶段 3: 决策树评估
**作用**：根据权重评分，自动判断能否实装

```bash
bash decision-tree.sh ./项目路径
```

**权重配置**：
| 检查项 | 权重 | 通过线 |
|--------|------|--------|
| 基础测试 | 30% | 必须 100% |
| 安全检查 | 30% | 必须 0 问题 |
| 代码质量 | 15% | ≥80 分 |
| 单元测试 | 15% | ≥80% |
| 构建检查 | 10% | 必须成功 |

**决策规则**：
- ✅ 可以实装：关键项通过 + 总分≥80
- ❌ 禁止实装：关键项失败 或 总分<80

**输出示例**：
```
========================================
  杰克实装决策树 v1.0
========================================

[1/5] 评分基础测试...
      ✅ 3/3 通过
[2/5] 评分安全检查...
      ✅ 0 个问题
[3/5] 评分代码质量...
      ✅ 得分 90
[4/5] 评分单元测试...
      ✅ 1/1 通过
[5/5] 评分构建检查...
      ✅ 1/1 成功

========================================
  总分：100 / 100
========================================

  ✅ 可以实装！

决策报告：./test-logs/decision-tree-20260309-123456.md
```

## 🎯 使用场景

### 场景 1：日常开发提交前

```bash
# 运行快速检查（1 分钟）
bash unified-gateway.sh --quick .

# 通过后提交
git add .
git commit -m "feat: 新功能"
```

### 场景 2：实装部署前

```bash
# 运行完整流程（5-10 分钟）
bash unified-gateway.sh --full .

# 根据决策树结果决定是否部署
# 如果通过：
./deploy.sh
```

### 场景 3：新项目启动

```bash
# 先查看历史错误案例
bash memory-review.sh .

# 避免历史错误，参考成功案例
```

### 场景 4：CI/CD集成

```yaml
# .github/workflows/test.yml
name: Test Framework

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Quick Check
        run: bash unified-gateway.sh --quick .

      - name: Decision Tree
        run: bash unified-gateway.sh --decision .
```

## ⚙️ 配置选项

### 环境变量

```bash
# 在脚本开头配置
export MAX_RETRIES=3           # 快速检查重试次数
export LOG_DIR="./test-logs"   # 日志目录
export PASSING_SCORE=80        # 决策树通过分数
```

### 决策树权重调整

编辑 `decision-tree.sh`：

```bash
declare -A WEIGHTS=(
    ["base_test"]=30      # 调整权重
    ["security"]=30
    ["code_quality"]=15
    ["unit_test"]=15
    ["build"]=10
)
```
