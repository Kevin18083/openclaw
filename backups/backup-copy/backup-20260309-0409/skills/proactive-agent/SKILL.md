# Proactive Agent Skill

## 功能特性

### 1. 心跳检查
- 定期检查系统状态
- 监控重要指标
- 自动报告异常

### 2. 定时任务
- 计划任务执行
- 周期性检查
- 自动提醒

### 3. 状态监控
- 系统资源监控
- 服务可用性检查
- 性能指标跟踪

## 基本配置

### 心跳配置
在 `HEARTBEAT.md` 中添加检查项：
```markdown
# 心跳检查清单

## 每日检查
- [ ] 检查未读邮件
- [ ] 查看日历事件
- [ ] 检查待办事项
- [ ] 监控系统状态

## 每小时检查
- [ ] 检查服务状态
- [ ] 监控资源使用
- [ ] 检查网络连接
```

### 定时任务配置
使用 `cron` 工具创建定时任务：
```javascript
// 示例：每天9点检查任务
{
  schedule: { kind: "cron", expr: "0 9 * * *", tz: "Asia/Shanghai" },
  payload: { kind: "agentTurn", message: "检查今天的任务和日程安排" },
  sessionTarget: "isolated"
}
```

## 使用示例

### 1. 创建每日提醒
```bash
# 创建每天早上8点的提醒
openclaw cron add --schedule '0 8 * * *' --message "早安！今天有什么计划？"
```

### 2. 监控服务状态
```bash
# 每30分钟检查一次服务状态
openclaw cron add --schedule '*/30 * * * *' --message "检查系统服务状态"
```

### 3. 定期报告
```bash
# 每天下午6点生成日报
openclaw cron add --schedule '0 18 * * *' --message "生成今日工作报告"
```

## 最佳实践

1. **适度检查**：不要设置过于频繁的检查，避免干扰
2. **重要优先**：优先监控关键系统和任务
3. **智能提醒**：根据时间和用户状态调整提醒方式
4. **记录历史**：保存检查记录以便分析

## 集成建议

- 与日历应用集成，提醒即将到来的事件
- 与任务管理工具集成，跟踪待办事项
- 与监控系统集成，实时报警
- 与通讯工具集成，发送通知
