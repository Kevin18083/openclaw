# 双记忆库使用指南

> 创建日期：2026-03-09 | 版本：v1.0

---

## 📋 概述

**双记忆库** = 错误记忆库 + 成功案例记忆库

**目的**：
- ✅ 记住每一个错误，不再犯同样的错
- ✅ 记住每一个成功，下次直接用
- ✅ 两者联动，形成完整知识体系

---

## 📚 两个记忆库

### 1. 错误记忆库

**文件**：`memory/error-memory.md`

**记录内容**：
- ❌ 测试失败的代码
- ❌ 运行出错的脚本
- ❌ 有 Bug 的实现
- ❌ 配置错误

**结构**：
```markdown
### 错误 001：错误名称

**发现日期**：YYYY-MM-DD

**测试命令**：出错的命令

**错误信息**：
```
错误输出
```

**根本原因**：
<!-- 分析结果 -->

**解决方案**：
<!-- 修复方法 -->

**学习收获**：
<!-- 学到了什么 -->

**如何避免**：
<!-- 预防措施 -->
```

---

### 2. 成功案例记忆库

**文件**：`memory/success-memory.md`

**记录内容**：
- ✅ 测试通过的代码
- ✅ 运行成功的脚本
- ✅ 经过验证的实现
- ✅ 最佳实践

**结构**：
```markdown
### 案例 001：成功案例名称

**成功日期**：YYYY-MM-DD

**成功内容**：
```bash
# 成功的代码
```

**成功关键**：

**使用场景**：
- 场景 1

- 错误案例 XXX：错误描述

- ✅ 直接用：XXX
- ⚠️ 调整：XXX
```

## 🔄 双记忆库联动流程

```
┌─────────────────────────────────────────┐
│         接到新任务                        │
└───────────────┬─────────────────────────┘
                ↓
┌─────────────────────────────────────────┐
│  1. 查成功案例记忆库                      │
│     - 有没有类似的成功案例？               │
│     - 有的话直接参考                      │
└───────────────┬─────────────────────────┘
                ↓
┌─────────────────────────────────────────┐
│  2. 查错误记忆库                          │
│     - 有没有类似的错误？                  │
│     - 有的话避免踩坑                      │
└───────────────┬─────────────────────────┘
                ↓
┌─────────────────────────────────────────┐
│  3. 写代码/做框架/配置                   │
│     - 参考成功案例                        │
│     - 避免错误案例                        │
└───────────────┬─────────────────────────┘
                ↓
┌─────────────────────────────────────────┐
│  4. 运行测试框架                          │
│     - 测试通过 → 记录到成功案例          │
│     - 测试失败 → 记录到错误案例          │
└───────────────┬─────────────────────────┘
                ↓
        ┌───────┴───────┐
        ↓               ↓
┌─────────────┐  ┌─────────────┐
│ 成功案例 +   │  │ 错误案例 +   │
│ 继续实装     │  │ 分析修复     │
└─────────────┘  └──────┬──────┘
                        ↓
               ┌────────────────┐
               │ 修复后重新测试  │
               └────────────────┘
```

## 💡 使用场景

### 场景 1: 写新脚本

```
1. 查成功案例记忆库
   → 找类似的脚本案例

2. 查错误记忆库
   → 避免类似的错误（如子 shell 变量、数组逗号）

3. 写脚本
   → 参考成功案例的结构
   → 避免错误记忆库的问题

4. 运行测试
   → 通过 → 记录到成功案例
   → 失败 → 记录到错误案例，分析修复
```

### 场景 2: 配置 CI/CD

```
1. 查成功案例记忆库
   → 找到"增量构建 CI"成功案例

2. 查错误记忆库
   → 看到"Thrift 验证子 shell 变量"错误
   → 看到"MySQL 端口检测不可靠"错误

3. 配置 CI
   → 参考成功案例的配置
   → 避免错误案例的问题

4. 运行测试
   → 通过 → 记录到成功案例
```

### 场景 3: 优化 VSCode 配置

```
1. 查成功案例记忆库
   → 找到"VSCode 大型企业项目配置"成功案例

2. 直接复用
   → TS Server 内存配置
   → watchOptions 配置
   → 文件关联配置

3. 测试
   → VSCode 不卡顿 → 记录成功
```

## 📖 实际例子

### 例子 1：Bash 数组配置

**错误记忆库 002**：
```markdown
### 错误 002：Bash 数组语法错误（多余逗号）

EXCLUDE_PATTERNS=(
  'apps/fornax/**',  # ❌ 多了逗号
  "packages/arch/semi-theme-hand01",  # ❌ 多了逗号
)
```

**成功案例记忆库 005**：
```markdown
### 案例 005：文件体积检查 Git Hook

EXCLUDE_PATTERNS=(
  'apps/fornax/**'   # ✅ 没有逗号
  "packages/arch/semi-theme-hand01"  # ✅ 没有逗号
)
```

**对比学习**：
- 错误案例：数组元素之间用逗号（错误）
- 成功案例：数组元素之间用空格或换行（正确）
- 下次写 Bash 数组，直接用成功案例的格式！

---

### 例子 2：子 shell 变量传递

**错误记忆库 001**：
```markdown
### 错误 001：Thrift 验证子 Shell 变量不生效

find ... | while read ...; do
  ERROR_FOUND=1  # ❌ 子 shell 中变量修改不生效
done
```

**成功案例记忆库 002**：
```markdown
### 案例 002：Go 多模块工作区测试

# 直接用父 shell 变量
for module in $modules; do
  go work use $module  # ✅ 直接修改
done
```

**对比学习**：
- 错误案例：`find | while` 在子 shell 中运行
- 成功案例：`for` 循环直接在父 shell 中运行
- 下次需要传递状态时，用临时文件或避免管道

## 🎯 最佳实践

### 1. 先查再写

```bash
# 写代码前
cat memory/error-memory.md    # 看有哪些错误要避免
cat memory/success-memory.md  # 看有哪些成功可参考
```

### 2. 测试通过再实装

```bash
# 用测试框架测试
./test-runner.sh ./your-script.sh

# 查看测试报告
cat test-logs/summary-*.log
```

### 3. 自动记录

```bash
# 测试框架会自动记录
# - 通过 → 成功案例
# - 失败 → 错误案例
```

### 4. 定期回顾

```bash
# 每周回顾一次
# - 新增了多少成功案例？
# - 新增了多少错误案例？
# - 有哪些错误可以避免？
```
