## 3. 12 类注释规范

### 3.1 什么是 12 类注释？

12 类注释是一套**标准化的代码注释模板**，适用于所有 JavaScript/Node.js 脚本。

**目的：**
- 让 AI（杰克/扎克）快速理解代码
- 让人类开发者快速上手
- 统一团队代码风格
- 便于长期维护

### 3.2 12 类注释完整清单

| 序号 | 类别 | 说明 | 必需内容 |
|------|------|------|----------|
| 1 | 功能说明 | 脚本能做什么 | 1-4 条功能列表 |
| 2 | 配置说明 | 有哪些配置项 | 常量/路径/参数 |
| 3 | 用法示例 | 怎么使用 | 命令行/代码调用 |
| 4 | 输入输出 | 输入什么/输出什么 | 数据类型/格式 |
| 5 | 依赖关系 | 依赖什么 | Node.js 版本/模块 |
| 6 | 常见问题 | 可能出什么错 | 错误 + 解决方案 |
| 7 | 设计思路 | 为什么这样设计 | 设计决策/权衡 |
| 8 | 修改历史 | 谁什么时候改了啥 | 日期 + 说明 |
| 9 | 状态标记 | 代码稳定程度 | ✅稳定/⚠️测试中 |
| 10 | 业务含义 | 业务上代表什么 | 业务场景说明 |
| 11 | 性能特征 | 性能怎么样 | 耗时/内存/瓶颈 |
| 12 | 安全考虑 | 有什么安全风险 | 权限/敏感数据 |

## 4. 缓存优化器使用

### 4.1 什么是缓存优化器？

**目标**: API 缓存命中率 32.6% → 50-70%

**好处：**
- ✅ 更快响应 - 命中缓存的请求响应速度提升 10-20%
- ✅ 节省 Token - 减少 15-25% 的 Token 消耗
- ✅ 降低成本 - 缓存读取比重新计算便宜

### 4.2 快速开始

```bash
cd C:\Users\17589\.openclaw\workspace
node zach-cache-optimizer.js test
```

### 4.3 使用方法

```javascript
const { generateOptimizedRequest } = require('./zach-cache-optimizer');

// 生成优化的请求
const request = generateOptimizedRequest(
  'code',           // 类别：basic, code, analysis, learn
  'func',           // 模板名
  {                 // 变量
    functionality: '计算斐波那契数列',
    language: 'JavaScript',
    inputs: 'n (数字)',
    outputs: '第 n 个斐波那契数'
  },
  'coder'          // 角色：default, coder, analyst, chat
);
```

## 9. 全自动化流程

### 9.1 自动化流程图

```
扎克提交任务 → Hook 自动触发 → 杰克 AI 初检 (DeepSeek) → 杰克 AI 深检
    ↓
自动归档 → 输出"杰克已阅" → 扎克向罗总汇报
```

### 9.2 自动化组件

| 组件 | 文件路径 | 功能 |
|------|----------|------|
| 提交 Hook | `~/.openclaw/.hooks/on-task-submit.js` | 扎克提交时自动触发 AI 检查 |
| 汇报 Hook | `~/.openclaw/.hooks/on-review-complete.js` | AI 审核完成自动触发汇报，输出"杰克已阅" |
| 汇报脚本 | `~/.openclaw/zack-report-to-luo.js` | 生成汇报文件并输出到控制台 |
| AI 检查脚本 | `workspace/jack-ai-review.js` | DeepSeek AI 检查（主用） |

### 9.3 AI 检查配置

| 项目 | 配置 |
|------|------|
| API 提供商 | DeepSeek |
| API 端点 | https://api.deepseek.com/v1 |
| 模型 | deepseek-chat |
| Token 成本 | ¥0.001/1K tokens（约¥0.0008/次） |
| 月度成本 | 约¥1.2（每天 50 次） |

### 9.4 自动化触发条件

| 事件 | 自动动作 |
|------|----------|
| 扎克提交任务 | Hook 自动触发 AI 初检 |
| 扎克提交内容 | 杰克 AI 开始初检 |
| 初检通过 | 自动进入深检 |
| 深检完成 | 自动归档 + 触发汇报 |
