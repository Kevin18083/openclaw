## 📚 什么是 12 类注释？

12 类注释是一套**标准化的代码注释模板**，适用于所有 JavaScript/Node.js 脚本。

**目的**：
- 让 AI（杰克/扎克）快速理解代码
- 让人类开发者快速上手
- 统一团队代码风格
- 便于长期维护

---

## 📋 12 类注释完整清单

| 序号 | 类别 | 说明 | 必需内容 |
|------|------|------|----------|
| 1 | 功能说明 | 脚本能做什么 | 1-4 条功能列表 |
| 2 | 配置说明 | 有哪些配置项 | 常量/路径/参数 |
| 3 | 用法示例 | 怎么使用 | 命令行/代码调用 |
| 4 | 输入输出 | 输入什么/输出什么 | 数据类型/格式 |
| 5 | 依赖关系 | 依赖什么 | Node.js 版本/模块 |
| 6 | 常见问题 | 可能出什么错 | 错误 + 解决方案 |
| 7 | 设计思路 | 为什么这样设计 | 设计决策/权衡 |
| 8 | 修改历史 | 谁什么时候改了啥 | 日期 + 说明 |
| 9 | 状态标记 | 代码稳定程度 | ✅稳定/⚠️测试中 |
| 10 | 业务含义 | 业务上代表什么 | 业务场景说明 |
| 11 | 性能特征 | 性能怎么样 | 耗时/内存/瓶颈 |
| 12 | 安全考虑 | 有什么安全风险 | 权限/敏感数据 |

 *   [输出示例第一行]

## 💡 各类注释写作要点

### 1. 功能说明
**要点**：用动词开头，简洁描述
```javascript
/**
 * 功能说明：
 * 1. 自动学习 - 自动学习内容并记录到记忆文件
 * 2. 知识管理 - 管理杰克的知识库文件
 * 3. 学习日志 - 记录每次学习的详细内容
 */
```

### 2. 配置说明
**要点**：列出所有可配置的常量
```javascript
/**
 * 配置说明：
 * - memoryPath: 杰克记忆目录路径
 * - learnLogPath: 学习日志文件路径
 * - knowledgePath: 知识库文件路径
 */
```

### 3. 用法示例
**要点**：给可直接运行的命令
```javascript
/**
 * 用法：
 *   node jack-auto-learn.js                    # 直接运行
 *   const JackAutoLearn = require('./jack-auto-learn')
 *   const jack = new JackAutoLearn()
 *   jack.learn('概念', '内容')
 */
```

### 4. 输入输出
**要点**：说清数据格式
```javascript
/**
 * 输入输出：
 *   输入：概念名称（字符串）、学习内容（字符串）
 *   输出：学习日志（Markdown 格式文件）
 */
```

### 5. 依赖关系
**要点**：写清版本和模块
```javascript
/**
 * 依赖关系：
 * - Node.js 14+
 * - fs, path (内置模块)
 */
```

### 6. 常见问题
**要点**：真实遇到的错误
```javascript
/**
 * 常见问题：
 * - 目录创建失败 → 检查磁盘空间和权限
 * - 文件写入失败 → 检查文件是否被占用
 * - 日志更新失败 → 确保文件未被其他进程占用
 */
```

### 7. 设计思路
**要点**：解释为什么，不是是什么
```javascript
/**
 * 设计思路：
 * 为什么设计自动学习系统？
 * - 杰克教扎克的过程中产生大量知识
 * - 需要自动记录和沉淀这些知识
 * - 避免重复学习相同内容
 * - 形成可复用的知识库
 */
```

### 8. 修改历史
**要点**：日期 + 版本 + 说明
```javascript
/**
 * 修改历史：
 * - 2026-03-09: 初始版本
 * - 2026-03-10: 添加 8 类注释
 * - 2026-03-11: 升级到 12 类注释
 */
```

### 9. 状态标记
**要点**：一目了然的稳定程度
```javascript
/**
 * 状态标记：
 * ✅ 稳定 - 生产环境使用
 * ⚠️ 测试中 - 功能未完全验证
 * 🚧 开发中 - 请勿使用
 */
```

### 10. 业务含义
**要点**：业务视角的价值
```javascript
/**
 * 业务含义：
 * - memoryPath: 杰克记忆目录，存储学习成果
 * - learnLogPath: 学习日志，记录学习历史
 * - knowledgePath: 知识库，结构化知识存储
 * - 教学相长：教扎克的过程也是杰克学习的过程
 */
```

### 11. 性能特征
**要点**：量化指标
```javascript
/**
 * 性能特征：
 * - 学习耗时：<50ms/次（文件写入）
 * - 内存占用：<5MB
 * - 日志增长：约 500-1000 字节/次学习
 * - 瓶颈：无明显瓶颈
 */
```

### 12. 安全考虑
**要点**：真实的风险点
```javascript
/**
 * 安全考虑：
 * - 学习日志不包含敏感业务数据
 * - 知识库文件权限设为 600
 * - 定期备份知识库（防止丢失）
 * - 不记录 API 密钥等敏感信息
 */
```

## 📌 常见问题 FAQ

### Q1: 12 类注释太多了，能否省略？
**A**: 不建议。每一类都有价值：
- 1-6 类：帮助理解代码
- 7-12 类：帮助长期维护

### Q2: 有些类别实在没内容怎么办？
**A**: 可以写"无明显瓶颈"、"不涉及敏感数据"等，但要说明原因。

### Q3: 注释太长会不会影响性能？
**A**: 不会。注释在运行时会被忽略，只在读取文件时有一次性消耗。

### Q4: 扎克需要写注释吗？
**A**: 需要！扎克修改代码时，注释帮助理解；扎克写新代码时，注释是模板。

### Q5: 为什么设计思路这么重要？
**A**: 因为代码会改，但设计决策的 rationale 帮助后人理解"为什么这样写"。
