## 🔍 决策树评分流程（5 项核心）

```
run_decision_tree(test_target)
│
├─ [1/5] 基础测试评分（权重 30%）⭐ 关键项
│   ├─ 环境检查（Node.js、Git）
│   ├─ 语法检查（bash -n）
│   └─ 通过率≥100% → +30 分
│
├─ [2/5] 安全评分（权重 30%）⭐ 关键项
│   ├─ 敏感文件扫描
│   ├─ SQL 注入检测
│   ├─ eval() 检测
│   └─ 0 问题 → +30 分
│
├─ [3/5] 代码质量评分（权重 15%）
│   ├─ TODO/FIXME 统计
│   ├─ 硬编码密钥检测
│   ├─ console.log 统计
│   └─ 分数≥90 → +15 分
│
├─ [4/5] 单元测试评分（权重 15%）
│   ├─ 运行 npm test / go test
│   ├─ 计算通过率
│   └─ 通过率≥90% → +15 分
│
├─ [5/5] 构建评分（权重 10%）
│   ├─ 运行 npm run build / go build
│   ├─ 构建后测试
│   └─ 100% 成功 → +10 分
│
└─ 计算总分
    ├─ 关键项（基础 + 安全）必须通过
    ├─ 总分≥80 → "✅ 可以实装！"
    └─ 否则 → "❌ 禁止实装！"
```
