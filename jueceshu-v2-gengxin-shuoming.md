## 📊 评分标准详解

### 1. 代码质量评分（90 分标准）

**扣分项**：

| 问题类型 | 扣分值 | 示例 |
|----------|--------|------|
| TODO/FIXME/HACK 标记 | -3 分/个 | `// TODO: 待优化` |
| 调试语句 | -8 分/个 | `console.log('debug')` |
| 大文件（>800 行） | -10 分/个 | 超过 800 行的文件 |
| 硬编码密钥/密码 | -15 分/个 | `password = "abc123"` |
| 空 catch 块 | -5 分/个 | `catch(e) {}` |
| 嵌套循环 | -5 分/个 | `for(){ for(){...} }` |

**示例计算**：
```
初始分数：100 分
- 3 个 TODO        → -9 分
- 2 个 console.log  → -16 分
- 1 个 1000 行文件   → -10 分
- 0 个硬编码密钥    → 0 分
- 0 个空 catch      → 0 分
- 1 个嵌套循环      → -5 分
───────────────────────────────
最终得分：60 分 ❌（低于 90 分要求）
```

---

### 2. 单元测试评分（90% 标准）

**评分规则**：

1. **有测试配置且通过**：基础分 100 分
2. **覆盖率检测**：
   - 覆盖率 ≥ 90%：保持 100 分
   - 覆盖率 60-89%：按比例折算
   - 覆盖率 < 60%：不及格

3. **测试文件检查**：
   - 有测试文件：+1 项通过
   - 无测试文件：❌ 不及格

**示例**：
```
项目 A:
- 测试通过 ✅
- 覆盖率 95% ✅
→ 得分：100 分 ✅

项目 B:
- 测试通过 ✅
- 覆盖率 70% ⚠️
→ 得分：100 * 70/100 = 70 分 ❌

项目 C:
- 无测试配置 ❌
→ 得分：0 分 ❌
```

---

### 3. 构建检查（增加测试环节）

**v1.0 流程**：
```
1. 运行构建
2. 检查构建成功
→ 完成
```

**v2.0 流程**：
```
1. 运行构建
2. 检查构建产物（dist/build/lib 目录）
3. 运行构建后测试
   - npm run test:build 或
   - npm test（普通测试）
4. 验证测试通过
→ 完成
```

**评分示例**：
```
项目 A:
- 构建成功 ✅
- 构建产物存在 ✅
- 构建后测试通过 ✅
→ 3/3 成功，得分：100 分 ✅

项目 B:
- 构建成功 ✅
- 构建产物存在 ✅
- 构建后测试未运行 ⚠️
→ 2/3 成功，得分：67 分 ❌
```

## 🎯 如何达到 90 分标准

### 代码质量建议

1. **减少 TODO 标记**
   ```bash
   # 查看 TODO 数量
   grep -r "TODO" --include="*.js" --include="*.ts" .

   # 尽快处理 TODO 项
   ```

2. **移除调试语句**
   ```bash
   # 查找 console.log
   grep -r "console.log" --include="*.js" --include="*.ts" .

   # 使用日志工具替代
   ```

3. **拆分大文件**
   - 单个文件不超过 800 行
   - 按功能模块拆分

4. **严禁硬编码密钥**
   ```bash
   # ❌ 错误
   const apiKey = "sk-1234567890"

   # ✅ 正确
   const apiKey = process.env.API_KEY
   ```

### 单元测试建议

1. **添加测试覆盖率**
   ```json
   // package.json
   {
     "scripts": {
       "test": "jest --coverage"
     }
   }
   ```

2. **目标覆盖率 90%**
   - 核心业务逻辑 100% 覆盖
   - 边界条件测试
   - 异常处理测试

3. **使用测试文件命名规范**
   ```
   *.test.js 或 *.spec.js
   __tests__/ 目录
   ```

### 构建测试建议

1. **添加构建后测试脚本**
   ```json
   {
     "scripts": {
       "build": "tsc",
       "test:build": "node dist/index.js --version"
     }
   }
   ```

2. **验证构建产物**
   - 检查 dist/ 或 build/ 目录
   - 验证生成的文件完整性
