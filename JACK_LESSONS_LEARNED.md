## 🔴 核心教训（按优先级）

### 1. 密钥管理要严格 【最高优先级】

**问题发现**:
- `.env` 文件被 Git 跟踪
- 包含 `TAVILY_API_KEY` 等敏感信息
- 多处代码/文档中出现真实密钥格式

**风险等级**: 🔴 严重

**学到的**:
```
❌ .env 文件绝不能放进 Git
❌ 真实密钥不能写在代码里
❌ 注释/文档/示例也不能放真实密钥
```

**以后怎么做**:
```
✅ 第一时间检查 .gitignore
✅ 密钥只用环境变量
✅ 示例用 "your_key_here" 代替
✅ 提交前运行密钥扫描
```

---

### 2. Git 仓库治理 【高优先级】

**问题发现**:
- `node_modules/` 被跟踪（8,666 个文件）
- `backups/` 被跟踪（3,453 个文件）
- 缺少 `.gitignore`
- 仓库体积 59MB+

**风险等级**: 🟠 高

**学到的**:
```
❌ node_modules 不该进 Git（是依赖，不是代码）
❌ backups 是运行数据，不是代码
❌ 大文件会让 clone/fetch 变慢
```

**以后怎么做**:
```
✅ 完善的 .gitignore（已创建）
✅ 定期 git status 检查
✅ 大文件用 Git LFS 或外部存储
✅ 代码与数据分离存储
```

---

### 3. Windows 路径兼容 【中优先级】

**问题发现**:
- `jack-daily-learning.js` 使用 `split('/')` 取文件名
- 多处 npm scripts 使用 Unix-only 命令
- Windows 路径是反斜杠 `\\`

**风险等级**: 🟡 中

**学到的**:
```
❌ split('/') 在 Windows 会出错
❌ Unix 命令 (sleep/tail/date) Windows 跑不了
❌ ~ 路径在 Windows 不认
```

**以后怎么做**:
```
✅ 路径处理统一用 path.basename()
✅ 脚本提供 .ps1 版本
✅ 用 cross-env/shx 等跨平台工具
✅ 避免硬编码路径分隔符
```

---

### 4. 依赖安全审计 【高优先级】

**问题发现**:
- `flatted <3.4.0` 高危漏洞（DoS）
- 依赖未及时更新

**风险等级**: 🟠 高

**学到的**:
```
❌ 依赖漏洞可能被利用
❌ 不更新依赖会积累风险
```

**以后怎么做**:
```
✅ 定期 npm audit --omit=dev
✅ 用 npm-check-updates 更新依赖
✅ CI 中加入安全检查
✅ 发现漏洞立即修复
```

---

### 5. 自动化检查的重要性 【中优先级】

**问题发现**:
- 人眼很难发现 12,411 个文件的问题
- 洛克一扫全部暴露
- 自动化工具效率极高

**风险等级**: 🟡 中

**学到的**:
```
✅ 自动化工具比人眼可靠
✅ 定期检查才能发现问题
✅ 预防比修复成本低
```

**以后怎么做**:
```
✅ 定期运行代码审计（每周/每月）
✅ CI 中加入安全检查
✅ 提交前自动 lint/test
✅ 密钥扫描纳入流程
```

---

### 6. 备份与隔离 【中优先级】

**问题发现**:
- 代码和数据混在一起
- 备份目录进了 Git
- Git 历史清理困难

**风险等级**: 🟡 中

**学到的**:
```
❌ 代码和数据不该混放
❌ 备份是运营数据，不是代码
❌ Git 历史清理很麻烦（需要提前预防）
```

**以后怎么做**:
```
✅ 代码/数据分离存储
✅ 备份用独立目录（.gitignore 保护）
✅ 敏感数据提前保护
✅ 不要等泄露再清理
```

## 📋 检查清单（以后每次都用）

### 提交前检查

```
□ .env 文件是否存在？→ 应该在 .gitignore 中
□ node_modules/ 是否被跟踪？→ 应该排除
□ backups/ 是否被跟踪？→ 应该排除
□ 代码中是否有真实密钥？→ 应该用环境变量
□ npm audit 是否通过？→ 应该 0 漏洞
□ Windows 路径是否兼容？→ 应该用 path 模块
```

### 定期审计（每月）

```
□ 运行全盘密钥扫描
□ 运行 npm audit
□ 检查 Git 跟踪的文件
□ 清理不需要的文件
□ 更新依赖版本
```

---

## 🎯 核心原则总结

| 原则 | 说明 |
|------|------|
| **密钥不出本地** | .env 永远不进 Git |
| **依赖要审计** | 定期 npm audit |
| **代码数据分离** | 备份/日志不进仓库 |
| **路径要兼容** | 用 path 模块，不用 split |
| **自动化优先** | 能自动的不要手动 |
| **预防胜于修复** | 提前保护比事后清理好 |
