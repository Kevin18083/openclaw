```

```

---

## 审查维度

### 1. 代码质量
- [ ] 函数命名清晰
- [ ] 函数长度 < 50 行
- [ ] 单一职责原则
- [ ] 无重复代码 (DRY)

### 2. 错误处理
- [ ] 有 try-catch 包裹
- [ ] 错误信息友好
- [ ] 日志级别正确 (DEBUG/INFO/WARN/ERROR)

### 3. 安全性
- [ ] 无硬编码密钥
- [ ] 输入验证完整
- [ ] 无 SQL 注入风险
- [ ] 无 eval 执行

### 4. 性能
- [ ] 无 N+1 查询
- [ ] 缓存使用合理
- [ ] 异步操作正确

### 5. 可维护性
- [ ] 有文件头注释
- [ ] 复杂逻辑有解释
- [ ] 配置与代码分离

---

## 审查流程

```
1. 扎克提交代码
   ↓
2. 自动运行审查脚本
   ↓
3. 杰克 AI 深度审查 (DeepSeek)
   ↓
4. 输出审查报告
   ↓
5. 扎克修复问题
   ↓
6. 重新提交验证
   ↓
7. ✅ 杰克已阅
```

## 命令集成

```bash
# 创建审查任务
node jack-review.js "让杰克检查代码审查"

# 提交审查结果
node jack-review.js submit "审查报告详情"

# 查看状态
node jack-review.js status
```
