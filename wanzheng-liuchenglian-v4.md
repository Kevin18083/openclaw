## 📋 21 个流程详细清单

### 第一阶段：基础测试（流程 1-5）

| # | 流程名称 | 函数名 | 检查内容 | 通过标准 | 预计耗时 |
|---|----------|--------|----------|----------|----------|
| 1 | **初步测试** | `run_test()` | 环境检查（Node.js、Git）、5 轮重试机制 | 5 轮内通过 | 30 秒 |
| 2 | **复查验证** | `verify_test()` | 连续 3 次通过同一测试 | 连续 3 次成功 | 30 秒 |
| 3 | **代码质量检查** | `code_quality_check()` | TODO/FIXME、硬编码密钥、console.log、未使用变量 | ≥90 分 | 1 分钟 |
| 4 | **安全检查** | `security_check()` | 敏感文件（.env、.pem）、SQL 注入、eval() 调用 | 0 问题 | 1 分钟 |
| 5 | **性能检查** | `performance_check()` | 嵌套循环 O(n²)、内存泄漏、异步阻塞 | 无严重问题 | 1 分钟 |

**阶段小计**：约 4 分钟

## 🔄 完整执行流程（代码级）

```

│

│   ├─ verify_test("环境复查", "node --version && git --version")

│   ├─ documentation_check("./目标路径")
│   └─ compatibility_check("./目标路径")
│
├─ [阶段 7/17] Git 提交规范检查
│   └─ git_commit_check("./目标路径")
│
├─ [阶段 8/17] 环境变量检查
```

## 📊 决策树评分流程（5 项核心）

```
run_decision_tree("./目标路径")
│
├─ [1/5] 评分基础测试 (权重 30%)
│   ├─ 环境检查 (Node.js、Git)
│   ├─ 语法检查 (bash -n)
│   └─ 通过率≥100% → +30 分
│
├─ [2/5] 评分安全检查 (权重 30%)
│   ├─ 敏感文件扫描
│   ├─ SQL 注入检测
│   ├─ eval() 检测
│   └─ 0 问题 → +30 分 (关键项)
│
├─ [3/5] 评分代码质量 (权重 15%)
│   ├─ TODO/FIXME 统计
│   ├─ 硬编码密钥检测
│   ├─ console.log 统计
│   └─ 分数≥90 → +15 分
│
├─ [4/5] 评分单元测试 (权重 15%)
│   ├─ 运行 npm test
│   ├─ 计算通过率
│   └─ 通过率≥90% → +15 分
│
├─ [5/5] 评分构建检查 (权重 10%)
│   ├─ 运行 npm run build
│   ├─ 构建后测试
│   └─ 100% 成功 → +10 分
│
└─ 计算总分
    ├─ 关键项 (基础 + 安全) 必须通过
    ├─ 总分 ≥ 80 → "✅ 可以实装！"
    └─ 否则 → "❌ 禁止实装！"
```
