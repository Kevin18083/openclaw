```
扎克遇到问题 → 在聊天框提问 → 杰克识别问题关键词
```

## 🎯 触发条件

扎克在聊天框或任务中提到以下关键词时自动触发：

| 类别 | 关键词 | 示例 |
|------|--------|------|
| 求助类 | 怎么办、怎么做、如何、怎么弄 | "这个报错怎么办" |
| 理解类 | 什么意思、是什么、为什么 | "这个参数是什么意思" |
| 错误类 | 报错、失败、错误、异常 | "缓存命中率上不去" |
| 确认类 | 对不对、可以吗、行不行 | "这样写对不对" |

## 🔧 实现方式

### 方式 1：聊天框关键词识别（推荐）

当扎克在网页端聊天框输入问题时：

```javascript
// on-question-submit.js
#!/usr/bin/env node

const QUESTION_KEYWORDS = [
  '怎么办', '怎么做', '如何', '怎么弄',
  '什么意思', '是什么', '为什么',
  '报错', '失败', '错误', '异常',
  '对不对', '可以吗', '行不行'
];

// 监听输入，检测问题
function detectQuestion(input) {
  for (const keyword of QUESTION_KEYWORDS) {
    if (input.includes(keyword)) {
      return true;
    }
  }
  return false;
}
```

### 方式 2：任务描述识别

扎克创建任务时，如果描述包含问题特征，自动触发杰克解答。

### 方式 3：错误日志监控

监控错误日志，发现扎克反复遇到的错误，主动推送解决方案。
